# Amax工具箱 (XIBIOS智控平台) 使用说明

## 一、系统概述

Amax工具箱是一个企业级服务器 BIOS/BMC 固件全生命周期管理平台。支持服务器机型管理、BIOS/BMC 固件版本追踪、文件上传下载、合规审计等功能，并内置软件中心用于发布和下载运维工具。

### 技术栈

- **前端**: React 19 + TypeScript + Vite 7 + Tailwind CSS 3
- **后端**: Express.js + better-sqlite3
- **UI 组件**: shadcn/ui (基于 Radix UI)

---

## 二、快速启动

### 环境要求

- **Node.js** 20+
- **npm**

### 启动方式

#### 方式一：一键启动脚本

```bash
chmod +x start.sh
./start.sh
```

- 前端地址: `http://localhost:3200`
- 后端 API: `http://localhost:3201`

#### 方式二：手动启动

```bash
# 安装依赖
npm install

# 同时启动前后端
npm run dev

# 或分别启动
npm run dev:web   # 仅前端 (Vite)
npm run dev:api   # 仅后端 API
```

#### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `WEB_PORT` | 3200 | 前端开发服务器端口 |
| `API_PORT` | 3201 | 后端 API 端口 |

## 四、功能模块说明

### 4.1 驾驶舱 (Dashboard) — 首页

进入系统后的首页，提供全局概览：

- **统计卡片**: 展示管理机型数、BMC 版本数、BIOS 版本数、待审核数、运维工具数
- **固件矩阵**: 展示最近添加的服务器机型及其 BIOS/BMC 版本状态
- **审计日志**: 显示最近的操作记录
- **快捷入口**: 可快速跳转到机型管理、软件中心、管理后台

### 4.2 机型库 (Models)

管理所有服务器机型，每个机型下可维护 BIOS 固件版本、BMC 固件版本、驱动程序和手册。

#### 添加机器

1. 点击页面右上角 **"添加机器"** 按钮
2. 在弹窗中填写信息：
   - **厂商**: 从品牌列表中选择
   - **主板/BB型号**: 如 `R750-2U`
   - **Amax型号**: 如 `AMX-750`
   - **平台类型**: Intel / AMD
   - **内存插槽数**: 默认 16
   - **CPU支持**: 如 `Intel Xeon Scalable`
   - **备注**: 可选的补充信息
3. 机器名称会自动由「厂商 + 主板型号 + Amax型号」拼接生成
4. 点击 **"添加机器"** 完成

#### 筛选机型

点击 **"筛选"** 按钮展开筛选面板，可按**厂商**或**状态**（已同步/待审核）过滤。

### 4.3 固件详情 (FirmwareDetail)

点击机型库中的任意机型卡片，进入固件详情页。该页面包含：

- **BIOS 固件**: 版本列表，支持上传新版本、下载、备注修改
- **BMC 固件**: BMC 固件版本管理，包含 BMC 类型、IP 地址等信息
- **驱动下载**: 相关驱动程序管理
- **说明书**: 相关手册文档管理

#### 上传固件

1. 在对应栏目中点击上传按钮
2. 填写版本号、发布日期、更新日志
3. 拖拽或选择固件文件进行上传
4. 系统自动计算 MD5 校验值和文件大小

### 4.4 镜像中心 (ISO Center)

管理系统 ISO 镜像文件，包含：

- 按分类浏览镜像（操作系统、驱动盘等）
- 镜像版本、架构、大小、MD5/SHA256 信息
- 下载次数统计

### 4.5 软件中心 (Software Hub)

运维工具的发布与下载平台。

#### 浏览工具

- 页面展示所有已发布的运维工具卡片
- 支持按**分类**筛选：诊断工具、自动化、安全工具、实用工具、监控工具
- 支持按**名称/描述/标签**搜索

#### 发布工具

1. 点击 **"发布工具"** 按钮
2. 填写工具信息：
   - 工具名称、描述
   - 分类：诊断工具 / 自动化 / 安全工具 / 实用工具 / 监控工具
   - 版本号（如 `1.0.0`）
   - 作者
   - 标签（逗号分隔）
   - README 使用说明
   - 上传工具文件
3. 发布后状态为 **"待审核"**，需管理员审核通过后方可下载

#### 下载工具

1. 点击工具卡片查看详情
2. 在详情弹窗中点击 **"下载工具"** 按钮
3. 已发布工具必须通过审核（状态为 "active"）才能下载

### 4.6 合规监控 (Audit)

审计日志页面，追踪所有操作记录：

- **操作类型**: 上传 (upload)、下载 (download)、更新 (update)、删除 (delete)、备注 (note_added)
- **记录内容**: 时间、操作用户、操作目标、详细信息
- 以时间线形式展示

### 4.7 管理后台 (Admin)

仅管理员及以上角色可访问。提供：

- 系统统计数据概览
- 固件审核（待审核版本确认）
- 系统设置

### 4.8 用户管理 (User Management)

仅超级管理员可访问。路径：`/admin/users`

#### 角色与权限

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 超级管理员 (super_admin) | 系统最高权限 | 所有模块的查看/创建/编辑/删除 |
| 普通管理员 (admin) | 可分配指定模块权限 | 由超级管理员指定可访问的模块 |
| 普通用户 (user) | 只读权限 | 仅查看，不可增删改 |

#### 模块权限说明

| 模块 | 标识 | 说明 |
|------|------|------|
| 驾驶舱 | dashboard | 首页仪表盘 |
| 机型库 | models | 服务器机型及固件管理 |
| 镜像中心 | isos | ISO 镜像管理 |
| 软件中心 | software | 运维工具发布下载 |
| 合规监控 | audit | 操作审计日志 |
| 管理后台 | admin | 系统管理功能 |

#### 创建用户

1. 进入 `/admin/users` 页面
2. 点击添加用户
3. 选择用户角色
4. 对于 admin 角色，可分配具体的模块访问权限
5. 设置用户状态（启用/禁用）

## 五、权限系统

系统采用 **RBAC（基于角色的访问控制）** 机制：

- **路由级权限**: 未登录用户仅可访问登录页；不同角色看到不同的导航菜单
- **模块级权限**: 每个模块独立控制查看/创建/编辑/删除权限
- **操作级权限**: 软件发布、固件上传等操作需对应权限

## 六、数据存储

- 系统使用 **SQLite** 数据库（文件位于 `data/amax.sqlite`）
- 用户数据、应用配置数据均持久化存储在数据库中
- 上传的文件存储在 `data/uploads/` 目录下
